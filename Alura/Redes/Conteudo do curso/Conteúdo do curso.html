Camadas de Redes

Dispositivo (Celular)

  • Aplicação
    • Dispositivos com acesso a rede
  • Transporte
    • Prepara (empacota) as mensagens para transportá-las através da rede
    • TCP é o principal protocolo da camada de transporte. A principal função da camada de transporte é justamente prover comunicação segura e eficiente entre processos de aplicação em dispositivos computacionais. No caso do website, estamos falando em comunicação entre os processos do navegador - executados em nosso computador - e processos do website - executados em um servidor.

Rede (Roteador)

  • Responsável por conectar dispositivos diferentes através de protocolos como o IP
  • IP
    • Adiciona o cabeçalho aos pacotes e roteamento
  • Física
    • Onde a mensagem é traduzida para sinais físicos para ser transportado pelo cabeamento onde será recebido em outra estrutura de rede que fará a conversão da mensagem do meio físico para o protocolo de comunicação e dará o caminho inverso da comunicação.

Estrutura

Um servidor hospeda uma aplicação Web e o outro servidor será configurado para ser um Servidor DNS.

Conceitos

Endereçamento IP
  • Endereço IP (Internet Protocol)

    Endereço universalmente reconhecido fora do dispositivo que pode ser atribuído pela rede interna e externa

    • IPV4
      • A versão 4 do endereçamento de IPs fornece 32 bits de dados para possibilitar a atribuição de IPs dentro dos parâmetros dessa versão. Isso significa que para cada octeto, temos 256 possibilidades (0-255) de atribuição numérica, ou seja, para cada octeto são 8 bits que somado são 32 bits (8 vezes 4).

        De 000.000.000.000 até 255.255.255.255 - São possíveis 4.228.250.625 de endereços de IP

        • Classes de IPV4
          • Classe A

            O primeiro octeto vai de 1-126 e a máscara de rede é 255.0.0.0

            Exemplo: 123.145.3.3

            Endereço de rede: 123.0.0.0

            Endereço de broadcast: 123.255.255.255

          • Classe B

            O primeiro octeto vai de 128-191 e a máscara de rede é 255.255.0.0

            Exemplo: 135.145.3.3

            Endereço de rede: 135.145.0.0

            Endereço de broadcast: 135.154.255.255

          • Classe C

            O primeiro octeto vai de 192-223 e a máscara de rede é 255.255.255.0

            Exemplo: 193.168.3.3

            Endereço de rede: 193.168.3.0

            Endereço de broadcast: 193.168.3.255

          • Classe D

            O primeiro octeto vai de 224-239

          • Classe D

            O primeiro octeto vai de 240-255

    • IPV6
      • A versão 6 é uma evolução para a versão 4 que nos fornece a possibilidade de 128 bits de dados para possibilitar a atribuição de IPs dentro dos parâmetros dessa versão. Isso significa que para cada octeto, temos 65.536 possibilidades (0-65.535) de atribuição numérica, ou seja, para cada octeto são 16 bits que somado são 128 bits (16 vezes 8).

        De 0000.0000.0000.0000.0000.0000.0000.0000 até 65535.65535.65535.65535.65535.65535.65535.65535 - São possíveis 3,40240830764391e38 de endereços de IP

    • IP público

      Identificação de dispositivos em uma rede externa

    • IP privado

      Identificação de dispositivos em uma rede interna

      Classe A - 10.0.0.0

      Classe B - 172.16.0.0 a 172.31.0.0

      Classe B - 192.168.0.0

    • IP estático

      Os endereços IP estáticos apresentam uma vantagem fundamental para operação de sistemas com servidores e dispositivos de acesso remoto: a manutenção de uma conexão permanente com qualidade e estabilidade. Dessa forma, quando desejamos hospedar um site em um servidor, por exemplo, é ideal adotarmos um endereço IP estático para que o serviço DNS não tenha problemas para realizar a conexão entre a URL e o servidor que armazena nosso site ou serviço. Podemos também usar endereços IP estáticos em impressoras para garantir estabilidade de conexão, facilidade de acesso e configuração.

    • IP dinâmico

      Os endereços IP dinâmicos são amplamente utilizados em redes para conexão de dispositivos de usuários, como notebooks, smartphones ou desktops. É muito comum conectarmos nossos smartphones em diferentes redes, não é mesmo!? Os endereços IP dinâmicos possibilitam que possamos nos mover entre diferentes redes sem qualquer demanda adicional de configuração, recebendo endereços IP automaticamente. Além disso, proporcionam uma grande economia de endereços IP, tornando o gerenciamento de endereços disponíveis em uma rede mais eficiente (evitando que endereços fiquem sem uso). Essa característica é essencial com a limitação dos endereços disponíveis no IPv4.

  • Endereço MAC

    Uma espécie de endereço interno da placa de rede do dispositivo.

Protocolos
  • Protocolo ARP
    • Permite identificar um endereço físico através de um endereço de IP
    • Este protocolo mantem uma relação entre o os endereços de IP e Mac das máquinas que estão envolvidas na comunicação
  • Protocolo ICMP

    Protocolo que controla o envio das mensagens e recepção do outro dispositivo em uma rede

  • DNS (Domain Name System)

    É um protocolo que memoriza os IPs de tal forma que sejam facilmente identificáveis (Como se fosse uma lista de telefone)

      Um servidor DNS é feito única e exclusivamente para a tradução de domínios para endereços de IP. Desse modo, sua função não é hospedar sites ou aplicações, mas sim permitir que as requisições cheguem aos locais corretos.

      • Exemplo
  • TCP

    O protocolo TCP encontra-se acima da camada onde o IP está localizado e ele é responsável por realizar o transporte das informações. Além disso, a camada de transporte possui também outro protocolo bastante conhecido, o UDP.

  • DHCP (Dynamic Host Configuration Protocol)

    Este protocolo atribui automaticamente um IP para um dispositivo que se conecte a rede para evitar que cada IP seja atribuído manualmente, o que pode ser um problema caso haja muitos dispositivos em uma rede.

    • Configurando o DHCP em uma rede

      Em uma rede onde 6 computadores estão conectados a um switch que por sua vez se conecta a um roteador, deve ser acessado o roteador através de um PC comum conectado por uma cabo console ao roteador.

      Uma vez conectado ao roteador, a configuração será feita pelo terminal da seguinte maneira:

Router(dhcp-config)$ network 193.168.3.1 255.255.255.0

## Configurar o Default Gateway do roteamento
Router(dhcp-config)$ default-router 193.168.3.1

## Sair da configuração de DHCP

Máscara de sub-rede

Todo e qualquer endereço de IP possui uma máscara correspondente. Nessa máscara é informado qual parte do IP faz parte da rede e qual faz parte do host.

Quer um exemplo? Um endereço IP, digamos de número 192.168.0.50, que possui uma máscara de número 255.255.255.0, possui como seu endereço de rede o número 192.168.0 e o seu host como 50.

Para que essa rede possa ser dividida, é necessário dividir o número da máscara. Desta forma, você pode utilizar valores diferentes que estejam dentro de 0 até 255.

Essa tarefa, após completa, recebe o nome de sub-rede.

Default Gateway
  • O default gateway é o endereço IP responsável por encaminhar pacotes para redes externas. Em um roteador, por exemplo, o Gateway padrão será o IP da porta do roteador em em que um computador está conectado e é a partir dessa porta que os pacotes serão encaminhados quando enviados para redes externas.

Equipamento

Hub
  • É um dispositivo de rede que atua no encaminhamento de mensagens entre dispositivos computacionais que estão conectados em suas portas.
    • Pros
      1. Simples e de baixo custo
    • Contras
      1. Menor privacidade pois uma mensagem enviada de um dispositivo específico conectado a ele para outro, será também enviada para todos os outros dispositivos na rede e isso pode ser uma questão importante com a segurança dos dados trafegados.
      1. Gasta mais tempo para se conectar ao dispositivo de origem pois antes do pacote sair do dispositivo de destino até o de origem, o Hub pode ter gasto tempo de conexão com outros dispositivos que não estavam diretamente envolvidos na comunicação e que mesmo assim receberá os pacotes, o que aumenta a possibilidade de haver um congestionamento dos pacotes.
Switch

Conecta dispositivos em uma rede de maneira que o pacote enviado de um dispositivo para o outro seja enviado apenas para os dispositivos envolvidos nessa comunicação, ou seja, diferentemente do Hub, o switch é capaz de identificar os dispositivos sabendo a origem do pacote e o destino. Assim o switch pode identificar exatamente o dispositivo de destino sem ter que enviar para todos os outros dispositivos na rede.

A maneira como o switch é capaz de identificar apenas os dispositivos envolvidos em uma comunicação (Origem/Destino) se dá pelos seguintes passos:

  1. O switch recebe um pacote de dados do dispositivo de origem e encaminha para todos os outros dispositivos da rede (Utilizando o protocolo ARP para descobrir o endereço de todos na rede através do endereço MAC).
  1. Dentre todos os dispositivos que receberam o pacote, o dispositivo de destino se identifica para o switch (mantido em uma memória interna) e a partir desse momento, todos os pacotes enviados do dispositivo de origem serão encaminhados apenas para o dispositivo de destino e não mais para os demais dispositivos da rede.

    Prós

    1. Mais seguro já que os pacotes enviados de um dispositivo para o outro ficaram limitados a essa conexão sem compartilhar os dados com os demais na rede

    Contras

    1. É possível eliminar a segurança do switch ao lotar a memória de endereços MACs em todas as portas. Isso fará com que o switch não tenha memória disponível para lembrar dos dispositivos de destino/origem e isso fará com que o switch passe a se comportar como um Hub, ou seja, enviar os dados para todos na rede e assim prejudicar a segurança.
      1. Ainda que seja possível atacar um switch, é recomendável configurar a opção de segurança na porta, para que fique limitada a receber uma quantidade máxima de endereços MAC.

Roteador

A principal função dos roteadores é interconectar redes, encaminhando seus pacotes de dados.

Para configurar um roteador é necessário o Uso de um PC conectado a ele através de uma cabo console em que em uma ponta é ligado no PC através do conector de um cabo de rede normal e no roteador se conecta através da outra ponta que é uma entrada serial.

Para ter acesso ao roteador pelo PC conectado, é utilizado o Putty que fornecerá uma maneira de acessar o terminar do roteador pelo PC

Cabos
  • Cabo console

    Cabo que permite acessar um dispositivo de rede por meio de um PC para que o PC possa manipular o dispo

  • Cabo direto

    Conecta um dispositivo a um equipamento de rede

  • Cabo Cruzado (Crossover)

    Conecta dois dispositivos diretamente, sem usar um dispositivo de rede como o hub para intermediar a comunicação
